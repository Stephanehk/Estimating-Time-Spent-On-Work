# Paper website — *Estimating Time Spent on Work Tasks*

Static, self-contained landing page advertising the paper. No build step.

## Run locally
```bash
cd paper_website
python3 -m http.server 8753
# open http://localhost:8753/index.html
```

## Structure
- `index.html` — page content (hero, abstract, 6 sections, footer/citation).
- `styles.css` — design system.
- `app.js` — interactive Figure 1 (occupation explorer) and Figure 4 (survival CDF),
  plus the full top-25 tables (Tables 5 & 6, sortable).
- `assets/figure2_method.png`, `figure3_wsj.png` — rendered from the PDF.
- `Estimating_Task_Time_Share_Release.pdf` — linked from the hero.
- `data/occupations.json` — per-occupation task time shares + rubric/simulation
  exposure flags for 876 occupations (Figure 1).
- `data/fig4_cdf.json` — 8 real survival-CDF series for Figure 4.
- `scripts/export_occupations.py` — regenerates `data/occupations.json`.
  Figure 4 data is regenerated by `../final_analysis/export_fig4_web_data.py`.

## Data provenance (no fabricated data)
- Time shares: `website/data/analysis_results/occupation_data2saveforwebsite.json`
  (predicted time-per-day per task ÷ 7-hour daily budget).
- Exposure flags: `data/onet_features_with_exposure.xlsx`
  (`hl_is_exposed` = rubric-based / Eloundou–HM&L; `alex_is_exposed` = simulation / Wan et al.).
- Figure 4 curves: regenerated via the repo's own `run_weighted_exposure_analysis` pipeline.
- Tables 1, 2, 3, 5, 6 transcribed from the paper.

Sanity check: Massage Therapists (default) reproduce Figure 1 — 42.9% of tasks
(≈44%) and 15.8% of working time (≈16%) exposed under the rubric measure.

## Deploy
Any static host (Amplify, GitHub Pages, S3, Netlify). Upload the `paper_website/`
directory as the site root.
