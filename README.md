# Estimating Time Spent on Work Tasks — reproduction code

This repository contains the minimal code and data needed to reproduce the experiments in
**"Estimating Time Spent on Work Tasks"**. It estimates, for nearly every U.S. occupation in
O*NET, the **share of working time spent on each task**, `s(t, o) = w(t, o) · E[f(t, o)]`, by
factoring time into (i) an expected task frequency from O*NET/CPS and (ii) a per-instance
time `w(t, o)` recovered by solving a linear program over LM-elicited pairwise time rankings.

Everything here runs **offline and deterministically**: the LM pairwise judgements are shipped
as a cache under `data/generated_data/`, so re-running the pipeline performs **no API calls**
and reproduces the released estimates exactly.

## Setup

```bash
pip install -r requirements.txt    # Python 3.12; numpy/pandas/scipy/matplotlib/openpyxl
```

All commands below are run **from the repository root**.

The figure scripts write their PNGs to `plots/` (regenerated, not committed). Figure 4 =
`fraction_occupations_at_or_above_fraction_tasks_exposed_simulation_based_vs_rubric_based[_moderate_exposure].png`;
Figure 5 = the `..._max_span_band[...]` variants (the shaded band is the Eq. 1 vs Eq. 2 objective).
`_moderate_exposure` is panel (a) (>25% threshold); the plain name is panel (b) (>50%).
(`simulation_based` and `rubric_based` in the filenames are the two task-level exposure measures
the paper compares.)

The second Table 2 column (mean τ agg. vs. indiv. and frac. significant) is also printed standalone by
`python3 validation/user_study_analysis/mean_kt_aggregate_vs_individual.py`.

### Main output: per-task estimate spreadsheet

`task_time_share_estimates.xlsx` (at the repo root; regenerated by
`data_formatting/export_per_task_estimates.py`) is the main product of this project. It maps every
`(occupation, task)` to our **predicted per-instance time**, the **expected daily frequency**, and
the **time spent per day estimate** (their product, `w(t,o)·E[f(t,o)]`), alongside O*NET importance
and the simulation-based / rubric-based exposure labels.

## Method entry points

- `find_time_per_task.py` — builds and solves the LP per occupation (default objective and
  `--find_max_span_weights`). Uses `--use_cps_constants` and `--prob_threshold 0.7`.
- `time_classification.py` — the LM pairwise time-preference classifier (reads/writes the cache).
- `utils/` — `constants.py`, `compute_onet_hours.py` (CPS-based per-SOC frequency constants),
  `llm_utils.py` (OpenAI Responses API logprob path; only used when recomputing).

## Reproducibility note (please read)

The figures/numbers in the table above are exactly reproducible. Two caveats:

- **Figure 2** is a hand-drawn method diagram and **Figure 3** is a Wall Street Journal
  screenshot — neither is generated by code. (Figure 1's *numbers* are reproduced by
  `analysis/figure1_massage_therapist_example.py`.)
- **Out of scope by design:** the Spearman correlation ρ=0.55 between our time shares `s(t,o)`
  and the Tamkin–McCrory time shares, and the full top-25 lists (Tables 5 & 6), are **not**
  reproduced here — they come from two scripts in the full research repo that were intentionally
  excluded from this minimal release. (The ρ=0.54 Spearman between our `w(t,o)` and their
  completion times *is* reproduced, by `time_share_correlations.py`.)

## Re-computing the LM rankings (optional)

Not required for any result above. To regenerate the pairwise judgements from scratch, set
`OPENAI_API_KEY` (model `gpt-5.2`) and delete the relevant
`data/generated_data/*task2time_preference_prob_threshold.json` caches before running
`find_time_per_task.py`.
