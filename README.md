# Estimating Time Spent on Work Tasks

Code and data for the paper **"Estimating Time Spent on Work Tasks"**. We estimate, for nearly
every U.S. occupation in O*NET, the **share of working time spent on each task**.

## Our estimates: `task_time_share_estimates.xlsx`

If you just want the estimates we produced, download **[`task_time_share_estimates.xlsx`](task_time_share_estimates.xlsx)**
at the root of this repository. It has one row per (occupation, task) — 17,525 tasks across 876
occupations — with the following columns (using the notation of the paper, where `t` is a task and
`o` an occupation):

| Column | Paper notation | Meaning |
|---|---|---|
| `Task ID` | — | O*NET task identifier |
| `Task Name` | — | O*NET task statement |
| `Occupation ID` | — | O*NET-SOC occupation code |
| `Occupation Name` | — | O*NET occupation title |
| `Time spent per day estimate (w(t,o) · E[f(t,o)])` | **s(t,o)** | Estimated **hours per day** a typical worker spends on task `t` in occupation `o`. This is our time-share quantity: within an occupation these values sum to a 7-hour workday, so dividing by 7 gives the task's **share of working time**. |
| `Time per single instance (w(t,o))` | **w(t,o)** | Estimated time (hours) to complete **one instance** of the task, recovered from the linear program over the language model's pairwise "which task takes longer" judgements. |
| `Expected daily frequency (E[f(t,o)])` | **E[f(t,o)]** | Expected **number of times per day** the task is performed, from O*NET frequency bins adjusted with CPS hours data. |

By construction `s(t,o) = w(t,o) · E[f(t,o)]` (column 5 = column 6 × column 7).

The file is regenerated by `python3 -m data_formatting.export_per_task_estimates` (reads the
per-occupation LP solution files in `data/generated_data/`).

## Reproducing the paper

Everything runs **offline and deterministically**: the language-model pairwise judgements are
shipped as a cache under `data/generated_data/`, so re-running the pipeline makes **no API calls**
and reproduces the released estimates exactly.

```bash
pip install -r requirements.txt    # Python 3.12; numpy/pandas/scipy/matplotlib/openpyxl
```

All commands below are run **from the repository root**.

| Paper artifact | Command | Expected result |
|---|---|---|
| **Time shares** `s(t,o)` (the core estimates) | `python3 -m find_time_per_task --model_name gpt-5.2 --n_trials 1 --prob_threshold 0.7 --use_cps_constants --find_max_span_weights --start_index 0 --end_index 879` | rewrites `data/generated_data/occupation_time_per_task_*_chosen_max_span_weigths.json`; the per-task time estimates are **byte-identical** to the shipped files (verified) |
| same, **uniform objective** (Eq. 2, App. C) | drop `--find_max_span_weights` | rewrites `..._chosen.json` |
| **`task_time_share_estimates.xlsx`** (the estimates above) | `python3 -m data_formatting.export_per_task_estimates` | writes `task_time_share_estimates.xlsx` at the repo root |
| **Figure 1** (Massage Therapists) | `python3 -m analysis.figure1_massage_therapist_example` | 44% of tasks exposed, 16% of working time exposed |
| **Figure 4** (exposure CCDFs) | `python3 -m analysis.figure4_exposure_share_cdf` | writes the 2 non-band PNGs to `plots/` (see below) |
| **Figure 5** (CCDFs with LP-objective band) | `python3 -m analysis.figure5_exposure_share_cdf_lp_objective_band` | writes the 2 `..._max_span_band` PNGs to `plots/` |
| **Table 2** (human-study agreement) | `python3 validation/user_study_analysis/table2_human_ranking_agreement.py --loop_all_socs` | Kendall τ (LP vs agg.) = 0.66 / 0.58 / 0.61 / 0.57 / 0.50; mean τ (agg. vs indiv.) = 0.50 / 0.67 / 0.65 / 0.56 / 0.46 |
| **Table 3** (top-25 set changes) | `python3 -m analysis.table3_top25_exposure_changes` | 11 of 25 occupations swap out |
| **Time-share correlations** (§Analysis: Tamkin–McCrory time + O*NET descriptors) | `python3 -m analysis.time_share_correlations --use_cps_constants --extra_details gpt-5.2_ONET_30.2_use_cps_constants_prob_thresh=0.7 --use_max_span_time_weights` | importance 504/876·0.47, importance×freq (core) 575/876·0.61, frequency 131/876·−0.32, Spearman 0.54 |
| **Feasible-set size** (§Validation, ℓ2 = 1.50) | `python3 validation/feasible_set_solution_spread.py --use_chosen_codes --extra_details gpt-5.2_ONET_30.2_use_cps_constants_prob_thresh=0.7` | mean biggest ℓ2 difference ≈ 1.50 — **very slow** (~1.8M LP solves over 879 occupations; runs for days). Stochastic: it samples 1,000 random directions per occupation, so the aggregate (mean 1.50) is reproduced only approximately. |

The figure scripts write their PNGs to `plots/` (regenerated, not committed). Figure 4 =
`fraction_occupations_at_or_above_fraction_tasks_exposed_simulation_based_vs_rubric_based[_moderate_exposure].png`;
Figure 5 = the `..._max_span_band[...]` variants (the shaded band is the Eq. 1 vs Eq. 2 objective).
`_moderate_exposure` is panel (a) (>25% threshold); the plain name is panel (b) (>50%).
(`simulation_based` and `rubric_based` in the filenames are the two task-level exposure measures
the paper compares.)

The second Table 2 column (mean τ agg. vs. indiv. and frac. significant) is also printed standalone by
`python3 validation/user_study_analysis/mean_kt_aggregate_vs_individual.py`.

## Repository layout

```
find_time_per_task.py        the LP time-share method (entry point)
time_classification.py       LM pairwise time-preference classifier (reads/writes the cache)
utils/                       constants, CPS frequency constants, LLM I/O
analysis/                    Figures 1/4/5, Table 3, time-share correlations, exposure helpers
data_formatting/             exposure loaders + the per-task estimate export
validation/                  Table 2 (human study) + the feasible-set spread
task_time_share_estimates.xlsx   the main output (per-task time shares)
data/
├── onet_data/               O*NET Task Ratings / Task Statements workbooks
├── crosswalks/              Census/SOC/O*NET crosswalks
├── cps_data/                CPS ASEC extract (per-SOC hours constants)
├── exposure_scores/         simulation_based_exposure.xlsx, rubric_based_exposure.xlsx
├── anthropic_index_data/    Tamkin & McCrory per-task time data
└── generated_data/          LM pairwise-ranking cache + per-occupation LP outputs + soc-code list
```

## Method entry points

- `find_time_per_task.py` — builds and solves the LP per occupation (default objective and
  `--find_max_span_weights`). Uses `--use_cps_constants` and `--prob_threshold 0.7`.
- `time_classification.py` — the LM pairwise time-preference classifier (reads/writes the cache).
- `utils/` — `constants.py`, `compute_onet_hours.py` (CPS-based per-SOC frequency constants),
  `llm_utils.py` (OpenAI Responses API logprob path; only used when recomputing).

## Reproducibility notes

- **Figure 2** is a hand-drawn method diagram and **Figure 3** is a Wall Street Journal screenshot —
  neither is generated by code. (Figure 1's *numbers* are reproduced by
  `analysis/figure1_massage_therapist_example.py`.)
- **Out of scope by design:** the Spearman correlation ρ=0.55 between our time shares `s(t,o)` and
  the Tamkin–McCrory time shares, and the full top-25 lists (Tables 5 & 6), are **not** reproduced
  here — they come from two scripts in the full research repo that were intentionally excluded from
  this minimal release. (The ρ=0.54 Spearman between our `w(t,o)` and their completion times *is*
  reproduced, by `time_share_correlations.py`.)
- **Re-computing the LM rankings** (not required for any result above): set `OPENAI_API_KEY` (model
  `gpt-5.2`) and delete the relevant `data/generated_data/*task2time_preference_prob_threshold.json`
  caches before running `find_time_per_task.py`.
